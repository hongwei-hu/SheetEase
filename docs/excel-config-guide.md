## 1. Excel 文件命名规则

| 规则 | 说明 |
|------|------|
| 扩展名 | 必须为 `.xlsx` |
| 首字母 | 必须**大写**（小写开头的文件会被跳过） |
| 不以 `~$` 开头 | Excel 临时文件会被自动跳过 |
| 工作表 Sheet 名 | 每个 Excel 文件只处理**第一个 Sheet**，Sheet 名称即为导出数据表的名称，需全局唯一 |

**合法示例**：`SampleData.xlsx`、`ItemConfig.xlsx`  
**跳过示例**：`sampleData.xlsx`（首字母小写）、`~$SampleData.xlsx`（临时文件）

## 2. 工作表结构总览

```
行号  用途
────────────────────────────────────────────
第 1  备注行        （策划注释，不导出）
第 2  表头行        （列的中文/英文说明，不导出）
第 3  类型行        （数据类型，可附加约束）
第 4  标签行        （ignore / required / 空）
第 5  字段名行      （C# 字段名，导出为 JSON key）
第 6  默认值行      （单元格为空时使用的默认值）
第 7+ 数据行        （实际数据，从第 7 行开始）
```

- **第 1 列**始终为主键列（id 列），不可忽略，不需要填写类型。  
- 所有行的**列数必须一致**。

---

## 3. 第 1 行：备注行

- 纯注释，供策划团队说明字段含义，**不会**导出到任何文件。
- 可以填写任意文本。

---

## 4. 第 2 行：表头行

- 每列的人类可读名称（如"道具名称"、"售价"等）。
- 会与备注行一起写入生成的 C# 脚本的 XML 注释（`/// <summary>` 中的说明）。
- **不导出**到 JSON。

---

## 5. 第 3 行：类型行

这是最关键的配置行。支持以下类型（大小写不敏感）：

### 基础标量类型

| 类型写法 | C# 映射 | 说明 |
|---------|---------|------|
| `int` / `int32` / `integer` | `int` | 整数 |
| `nnint` | `int` | 非负整数，等价于内置 `int{nonnegative}` |
| `pint` | `int` | 正整数，等价于内置 `int{positive}` |
| `float` / `double` | `float` | 浮点数 |
| `nnfloat` | `float` | 非负浮点数，等价于内置 `float{nonnegative}` |
| `pfloat` | `float` | 正浮点数，等价于内置 `float{positive}` |
| `string` / `str` | `string` | 字符串 |
| `bool` / `boolean` | `bool` | 布尔值（填 `true`/`1` 或 `false`/`0`） |

### 集合类型

| 类型写法 | 说明 |
|---------|------|
| `list(int)` | 整数列表，单元格内用逗号分隔，允许重复元素 |
| `list(float)` | 浮点数列表 |
| `list(string)` | 字符串列表 |
| `list(bool)` | 布尔值列表 |
| `list(enum(EnumName))` | 枚举列表 |
| `unilist(int)` | 整数唯一性列表，所有元素必须互不重复，导出时若有重复则报错 |
| `unilist(float)` | 浮点数唯一性列表 |
| `unilist(string)` | 字符串唯一性列表 |
| `unilist(enum(EnumName))` | 枚举唯一性列表 |
| `dict(string, int)` | 键值对字典，单元格内格式 `key1:val1,key2:val2` |
| `dict(string, float)` | 字符串键、浮点值字典 |
| `dict(string, string)` | 字符串键值字典 |
| `dict(int, string)` | 整数键、字符串值字典 |

### 枚举类型

```
enum(MyEnumName)
```

枚举类型必须先在枚举定义表中声明，详见[第 14 节](#14-枚举类型定义)。

### 自定义类型

形如 `Namespace.TypeName`（含 `.` 的标识符），需提前注册自定义解析器，详见[第 17 节](#17-自定义类型)。

---

## 6. 第 4 行：标签行

| 标签值 | 含义 |
|--------|------|
| （空） | 普通可选字段，单元格为空时使用默认值或空值 |
| `ignore` | 忽略此列，**不**导出到 JSON 和 C# |
| `required` | 必填字段，若单元格为空且无默认值，导出时**报错** |

> 主键列（第 1 列）不需要填写标签。

---

## 7. 第 5 行：字段名行

- 字段名需符合 **C# 标识符规范**：
  - 仅含字母、数字、下划线
  - 不以数字开头
  - 不与 C# 关键字冲突
- 每张表内字段名必须**唯一**。
- 支持以下特殊前缀（详见[第 11 节](#11-字段名前缀标记)）。

---

## 8. 第 6 行：默认值行

- 当对应数据单元格为空时，使用此行的值进行填充。
- 填写格式与数据行相同（同类型）。
- 若为空且字段也为空，则根据类型使用零值：
  - `int` → `0`，`float` → `0.0`，`string` → `""`，`bool` → `false`
  - `list(...)` → `[]`，`dict(...)` → `{}`

---

## 9. 第 7 行起：数据行

- 从第 7 行开始，每行代表一条记录。
- **第 1 列**为主键（见[第 10 节](#10-主键策略)）。
- 遇到**完全空行**会跳过，不报错。

### list 类型单元格格式

用英文逗号 `,` 分隔多个值：
```
1,2,3
苹果,香蕉,橙子
true,false,true
```

### dict 类型单元格格式

用英文冒号 `:` 分隔键值，逗号 `,` 分隔条目：
```
atk:100,def:50,spd:80
fire:1.5,water:0.8
```

---

## 10. 主键策略

导出工具支持三种主键策略，由第 5 行（字段名行）的写法决定：

### 策略一：普通 int 主键（默认）

- 第 1 列（字段名行）填写普通字段名（如 `id`）
- 第 1 列数据填整数，要求全局唯一
- **建议**字段名命名为 `id`

```
字段名行：  id         name         attack
数据行：    1          "火焰剑"      100
数据行：    2          "冰霜弓"      80
```

### 策略二：字符串枚举主键

- 第 1 列数据填字符串名称（合法 C# 枚举名）
- 工具会自动生成整型自增 id 和对应枚举文件（`{表名}Keys.cs`）
- 枚举名命名规范同 C# 标识符

```
字段名行：  keyName      description
数据行：    Item_Sword   "火焰剑"
数据行：    Item_Bow     "冰霜弓"
```

### 策略三：组合 int 主键

- **字段名行**中**前两列**（第 1、2 列）分别写 `key1:真实名` 和 `key2:真实名`
- 这两列**类型行**必须均为 `int`
- 组合规则：`combined_id = key1 * 46340 + key2`，结果需在 int32 范围内

```
字段名行：  key1:groupId    key2:subId    name
类型行：    int             int           string
数据行：    1               0             "第一组第0项"
数据行：    1               1             "第一组第1项"
```

## 11. 字段名前缀标记

字段名行中的前缀不会出现在最终字段名中，仅用于标注特殊行为。

### `[Sheet/Field]` —— 跨表引用检查

格式：`[目标表名/目标字段名]字段名` 或 `[目标表名]字段名`（省略字段名时默认为 `id`）

导出时检查此字段的值在目标表的指定字段中是否存在，不存在则报警告。

```
字段名行：  id    [ItemConfig/id]itemRef    name
```

### `[Asset]` 或 `[Asset:ext]` —— 资源路径标记

格式：`[Asset]字段名` 或 `[Asset:png]字段名`

标注此字段存储资源路径（支持指定扩展名），信息写入 JSON 的 `_meta.fields` 区域，供运行时工具使用。

```
字段名行：  id    [Asset:png]iconPath    name
```

## 12. 类型注解详解

### bool 类型填写规范

| 单元格填写 | 解析结果 |
|-----------|---------|
| `true` 或 `1` | `true` |
| `false` 或 `0` | `false` |
| 空 | `false` |

### list 嵌套说明

- `list(enum(QualityType))`：枚举列表，值需为枚举名称
- 暂不支持 `list(list(...))`（不允许超过 3 层嵌套）

### dict 键值类型说明

- 键支持：`int`、`string`
- 值支持：`int`、`float`、`string`、`bool`、`enum(EnumName)`
- 键必须唯一，重复键会覆盖

## 13. 约束配置（新功能）

在类型注解末尾附加 `{约束名:值, ...}` 可以为字段添加值范围或格式约束。导出时若某行数据违反约束，导出会**报错并中断**，精确指出违规的表名、字段名和行号。

**格式**：`类型{约束1, 约束2:值, 约束3:值}`

多个约束用逗号分隔，布尔约束（无值的）直接写名称，数值/字符串约束写 `名称:值`。

---

### int / float 数值约束

| 约束 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `min:数值` | 数字 | 最小值（含边界）| `int{min:0}` |
| `max:数值` | 数字 | 最大值（含边界）| `int{max:9999}` |
| `min_exclusive:数值` | 数字 | 最小值（不含边界，即严格大于）| `float{min_exclusive:0}` |
| `max_exclusive:数值` | 数字 | 最大值（不含边界，即严格小于）| `float{max_exclusive:1.0}` |
| `nonzero` | 布尔 | 不允许为 0 | `int{nonzero}` |
| `positive` | 布尔 | 必须为正数（> 0）| `int{positive}` |
| `nonnegative` | 布尔 | 必须为非负数（>= 0）| `int{nonnegative}` |

如果这个约束是字段的固定语义，也可以直接把类型写成 `nnint`、`nnfloat`、`pint` 或 `pfloat`，不再需要重复追加对应的 `{nonnegative}` / `{positive}`。

**组合示例：**

```
int{min:1, max:9999}          → 值在 1~9999 之间（含两端）
float{min:0.0, max:1.0}       → 概率值 0.0~1.0
int{positive}                  → 必须是正整数（> 0）
float{min_exclusive:0, max_exclusive:1.0}  → 不含 0 和 1.0 的开区间
```

---

### string 字符串约束

| 约束 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `nonempty` | 布尔 | 不允许空字符串 | `string{nonempty}` |
| `minlen:数值` | 整数 | 最短字符数 | `string{minlen:1}` |
| `maxlen:数值` | 整数 | 最长字符数 | `string{maxlen:64}` |
| `len:数值` | 整数 | 固定字符数 | `string{len:8}` |
| `pattern:"正则"` | 字符串 | 全匹配正则表达式 | `string{pattern:"^[A-Za-z_][A-Za-z0-9_]*$"}` |

**组合示例：**

```
string{nonempty, maxlen:50}
string{minlen:2, maxlen:20}
string{pattern:"^[0-9]{4,8}$"}          → 4~8 位纯数字
string{nonempty, pattern:"^[A-Z_]+$"}    → 非空且仅含大写字母和下划线
```

> **注意**：`pattern` 值需用英文引号包裹，使用 Python `re.fullmatch` 语义（整串匹配）。

---

### list 列表约束

| 约束 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `nonempty` | 布尔 | 不允许空列表 | `list(int){nonempty}` |
| `minlen:数值` | 整数 | 最少元素数 | `list(string){minlen:1}` |
| `maxlen:数值` | 整数 | 最多元素数 | `list(int){maxlen:10}` |

**组合示例：**

```
list(int){nonempty, maxlen:5}
list(string){minlen:1, maxlen:3}
```

**关于唯一性约束**：

如果需要确保列表元素不重复，请直接使用 `unilist(...)` 类型而不是 `list(...){unique}`。`unilist` 是专门为不重复元素设计的基础类型，例如：

```
unilist(int)                    → 没有重复的整数列表
unilist(string){maxlen:20}      → 没有重复的字符串列表，长度最多 20
```

`unilist` 在导出时会严格检查重复元素，若有重复则直接报错，比添加 `{unique}` 约束更明确清晰。

---

### dict 字典约束

| 约束 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `nonempty` | 布尔 | 不允许空字典 | `dict(string,int){nonempty}` |
| `minsize:数值` | 整数 | 最少条目数 | `dict(string,int){minsize:1}` |
| `maxsize:数值` | 整数 | 最多条目数 | `dict(string,int){maxsize:5}` |

**组合示例：**

```
dict(string,int){nonempty, maxsize:10}
dict(string,float){minsize:1}
```

以下展示一张典型 Excel 表的类型行配置（第 3 行）：

## 14. 枚举类型定义

枚举通过单独的 Excel 文件进行集中定义（或在工具配置中指定枚举路径）。

工具在第一阶段会扫描所有 Excel 文件，收集枚举定义，然后在第二阶段供类型注解 `enum(EnumName)` 引用。

### 枚举定义表格式

| 列 | 用途 |
|----|------|
| 第 1 列 | 枚举名（如 `QualityType`） |
| 第 2 列 | 枚举成员名（如 `Common`、`Rare`） |
| 第 3 列 | 枚举整数值（可选，留空则自动从 0 递增） |
| 第 4 列 | 枚举成员注释（可选） |

## 15. 引用检查

在字段名行使用 `[目标表/目标字段]字段名` 语法，导表时会验证该字段的值在目标表的指定字段中存在。

- `[ItemConfig]questItem`：验证 `questItem` 的值在 `ItemConfig` 表的 `id` 字段中存在
- `[ItemConfig/name]questItem`：验证 `questItem` 的值在 `ItemConfig` 表的 `name` 字段中存在

**跳过检查的特殊值：**
- `int` 类型：值为 `0` 时跳过（表示"无引用"）
- `string` 类型：值为空字符串 `""` 时跳过

字典类型字段**不支持**引用检查（会打印警告并跳过）。

---

## 16. 资源标记

在字段名行使用 `[Asset]字段名` 或 `[Asset:扩展名]字段名`：

- 不触发额外的运行时校验逻辑（与旧版本不同）
- 标记信息写入导出 JSON 的 `_meta.fields` 区域，供运行时工具或构建管线使用
- 支持指定资源扩展名：`[Asset:png]`、`[Asset:prefab]` 等

```
字段名行：  id    [Asset:png]icon    [Asset:prefab]modelPath
```

## 17. 自定义类型

自定义类型写法：`Namespace.TypeName`（含 `.` 的类型标识符）。

**注册方式**（在 Python 代码中）：

```python
from ExcelExportTool.parsing.data_processing import custom_type_registry

def parse_my_type(raw: str):
    # raw 为单元格字符串，返回任意 JSON 可序列化对象
    parts = raw.split("#")
    return {"x": parts[0], "y": parts[1]}

custom_type_registry.register("Game.Vector2Int", parse_my_type)
```

未注册的自定义类型会通过通用回退解析器处理（按 `#` 分割为 `segments` 列表）。

## 19. 常见错误与排查

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `发现重复字段` | 同一张表中字段名行出现重复字段名 | 修改重复的字段名 |
| `主键重复` | 两行的主键（第1列）相同 | 检查并修改重复主键 |
| `非法字段名` | 字段名不符合 C# 标识符规范 | 改为字母/数字/下划线，不以数字开头 |
| `表头格式错误` | 前 6 行列数不一致 | 确保所有行列数相同 |
| `required 但值为空` | 标记为 required 的字段无值且无默认值 | 补充数据或添加默认值 |
| `约束违规` | 字段值不满足类型行中配置的约束 | 修正数据或调整约束范围 |
| `未注册的自定义类型` | 使用了未注册的自定义类型 | 在代码中注册该类型的解析器 |
| `Sheet 命名冲突` | 多个 Excel 文件的第一个 Sheet 同名 | 修改 Sheet 名称保证全局唯一 |
| `枚举名不合法` | 字符串主键表中出现不符合 C# 规范的枚举名 | 修正第 1 列的字符串值 |
| `工作表命名冲突` | 两个 Excel 的第一个 Sheet 同名 | 保证所有 Excel 的第一个 Sheet 名唯一 |

## 20. 配置实践建议日志

导表时，工具会基于当前数据状态输出绿色的“建议”日志，用于帮助你逐步优化类型配置。该功能为**建议型提示**：

- 不会修改 Excel 配置
- 不会中断导表流程
- 不会影响导出结果

### 首批建议规则（数值类型）

- 当标量 `int` 字段所有有效值都 `> 0` 时，建议改为 `pint`
- 当标量 `int` 字段所有有效值都 `>= 0` 且包含 `0` 时，建议改为 `nnint`
- 当标量 `float` 字段所有有效值都 `> 0` 时，建议改为 `pfloat`
- 当标量 `float` 字段所有有效值都 `>= 0` 且包含 `0` 时，建议改为 `nnfloat`

> 说明：仅对普通 `int/float` 标量字段给出建议；已经使用 `pint/nnint/pfloat/nnfloat` 或显式约束 `positive/nonnegative` 的字段不会重复提示。

### 规则元数据格式（可扩展）

建议规则统一使用如下元数据字段，便于后续按类别扩展、过滤或配置开关：

- `rule_id`：规则唯一标识
- `name`：规则名称
- `category`：规则分类（如 `constraint-tightening`）
- `target`：适用目标（如 `scalar:float`、`list:*`）
- `output`：输出类型（如 `constraint`、`type`）
- `description`：规则说明

### 本轮新增的两条收束规则

- `numeric.float.probability.constraint`：当 `float` 字段所有有效值都在 `[0,1]` 区间时，建议配置为 `float{min:0,max:1}`
- `list.unique.constraint`：当 `list(...)` 字段样本中始终无重复元素时，建议在类型上追加 `{unique}`
